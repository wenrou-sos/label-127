# 干洗店会员消费记录查询与统计分析面板系统 — 产品需求文档（PRD）

## 1. 产品概述

本系统是一套面向干洗连锁门店的会员消费记录查询与统计分析面板，同时服务于**收银端（门店员工）**与**会员端（C 端会员）**两个角色。系统解决会员身份核验、消费明细追溯、储值/套餐扣款透明化、余额变动审计以及异常盗刷预警等核心业务痛点，目标是提升门店收银效率、增强会员信任感并降低资金风险。

- **目标用户**：干洗店收银员、店长、C 端会员本人
- **产品价值**：统一消费数据视图，收银秒级核验会员，会员自助查账并导出，异常消费自动触发短信验证兜底

## 2. 核心功能

### 2.1 用户角色

| 角色 | 登录方式 | 核心权限 |
|------|----------|----------|
| 收银员（门店端） | 工号密码 / 直接进入工作台 | 会员查询、查看会员信息卡与消费明细、处理异常消费预警、触发短信验证 |
| 会员（C 端） | 手机号 + 短信验证码登录 | 查询个人消费流水、余额变动、日期筛选、导出消费记录 |

### 2.2 功能模块

**收银端（Cashier）**
1. **会员身份核验**：手机号输入查询 + 模拟刷卡两种方式
2. **会员信息卡片弹窗**：姓名、等级、储值余额、套餐余次、本月累计消费、等级升级进度条（含具体升级提示文案）
3. **消费明细列表**：按时间倒序，含消费日期、门店、消费项目明细与合计、扣款方式、扣款金额
4. **安全预警中心**：异常消费行为监测看板（同日频繁大额），异常详情与短信验证流程触发

**会员端（Member）**
1. **登录认证**：手机号 + 短信验证码
2. **消费流水查询**：个人消费记录列表
3. **余额变动记录**：充值 / 消费 / 退款 / 过期 四类变动明细
4. **日期范围筛选**：自定义起止时间区间
5. **消费记录导出**：导出为 CSV 格式

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 收银端工作台 | 会员核验栏 | 手机号输入框 + 刷卡按钮，支持回车查询，带防抖 |
| 收银端工作台 | 会员信息卡弹窗 | 弹窗展示会员基本信息、储值/套餐、本月消费、升级进度条，含「关闭/查看明细」操作 |
| 收银端工作台 | 消费明细列表 | 时间倒序列表，展示日期、门店、项目明细（含合计）、扣款方式徽标、金额，支持分页 |
| 收银端工作台 | 安全预警侧栏 | 实时异常监测卡片列表，点击查看详情并触发短信验证流程弹窗 |
| 收银端工作台 | 统计概览卡 | 本日消费笔数、本日营业额、本月新增会员、待处理预警数 |
| 会员端登录页 | 登录卡片 | 手机号输入 + 验证码输入 + 倒计时获取验证码 + 登录按钮 |
| 会员端首页 | 消费流水 | 消费记录卡片列表，含门店、项目、金额、扣款方式 |
| 会员端首页 | 余额变动 | 变动记录时间轴，区分充值/消费/退款/过期四类，带收支金额与变动后余额 |
| 会员端首页 | 日期筛选器 | 起止日期选择 + 快捷区间（近7天/本月/近30天） + 导出按钮 |

## 3. 核心流程

**流程一：收银端会员核验与查看消费**
收银员进入工作台 → 输入手机号或点击刷卡 → 系统校验会员 → 弹出会员信息卡 → 查看升级进度 → 展开消费明细列表 → 如检测到异常消费则侧栏高亮预警 → 点击预警触发短信验证 → 验证通过后继续操作。

**流程二：会员端查账与导出**
会员登录页输入手机号 → 获取并输入短信验证码 → 进入首页查看消费流水 → 切换至余额变动模块 → 选择日期范围筛选 → 点击导出 CSV 下载文件。

## 4. 用户界面设计

### 4.1 设计风格

- **设计方向**：「精致织物护理（Premium Fabric Care）」— 采用编辑式杂志风格的暖米色背景搭配深森林绿主色与暖金点缀，传递干净、专业、高端护理的质感，区别于常见的蓝紫科技风后台。
- **主色**：深森林绿 `#1E4D3B`（主操作/品牌）；**辅色**：暖金 `#C9A24B`（会员等级/进度）；**强调色**：砖红 `#B5462E`（预警/扣款警示）
- **背景**：暖米色 `#F7F3EC`；**卡片**：纯白带暖色阴影 `rgba(30,77,59,0.06)`
- **字体**：标题用衬线字体 `Fraunces` + `Noto Serif SC`（编辑质感）；正文用 `Manrope` + `Noto Sans SC`；数据用等宽数字 `tabular-nums`
- **按钮**：主按钮深绿实心圆角，次按钮描边，危险操作砖红；圆角统一 `0.75rem`
- **布局**：桌面端左侧固定侧栏导航 + 顶部品牌栏 + 主内容区卡片网格；会员端移动优先单列卡片流
- **图标**：`lucide-react` 线性图标，统一描边 1.75px
- **动效**：页面加载用 framer-motion 做错峰渐显，弹窗缩放淡入，进度条与数字滚动动画，预警卡片脉冲呼吸

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素（样式 / 布局 / 颜色 / 字体 / 动效） |
|----------|----------|----------------------------------------------|
| 收银端工作台 | 会员核验栏 | 顶部大尺寸搜索框，深绿描边，刷卡按钮带 NFC 图标，回车触发查询动画 |
| 收银端工作台 | 统计概览卡 | 4 列网格，白卡片 + 暖金数字 + 图标角标，数字滚动动效 |
| 收银端工作台 | 会员信息卡弹窗 | 居中弹窗，左侧会员头像与等级徽章，右侧储值/套餐/本月消费数字，底部进度条带文案 |
| 收银端工作台 | 消费明细列表 | 白卡片表格，扣款方式彩色徽标，项目明细折叠展开，分页器 |
| 收银端工作台 | 安全预警侧栏 | 右侧固定栏，异常卡片砖红描边脉冲呼吸，详情按钮触发短信验证弹窗 |

### 4.3 响应式

- 桌面优先（`lg` 断点）为收银端主战场，宽屏三栏布局；`md` 收银端统计卡降为两列、侧栏折叠为抽屉；`sm` 会员端单列、收银端隐藏侧栏改为底部预警条。
- 触屏优化：按钮最小高度 44px，表单输入禁用缩放，模态弹窗移动端底部抽屉式弹出。

### 4.4 数据可视化

- 统计概览采用大号数字 + 同比/环比小字；升级进度条为渐变填充条 + 文案标注；异常预警采用脉冲色块；会员端余额变动采用时间轴而非折线图以保证移动端可读性。所有图表/进度条均纯 CSS + SVG 实现，避免重型图表库以保性能。
